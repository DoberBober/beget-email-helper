# Работа с почтовыми ящиками на Beget.

- Отображать созданные почтовые ящики.
- Отображать названия папок в почтовых ящиках.
- Массово удалять письма.
- Массово сбрасывать пароли.

## Установка.

- Создаём виртуальное окружение `python3 -m venv venv`.
- Активируем `source venv/bin/activate`.
- Устанавливаем пакеты `pip install -r req.txt`.
- Заносим данные в `.env` и `accounts.json` (при необходимости).
- Запускаем нужный скрипт.

## 1. Список почтовых ящиков на аккаунте/домене.

Запускается командой `python mails_list.py --domains domain.ru,domain2.ru`.

- В параметре `domain` передаётся список доменов через запятую. Если параметр не передан, анализируются все домены на аккаунте.

После запуска в папке `results/mails_list` появится файл со списком почтовых ящиков на домене.

## 2. Список папок в почтовом ящике.

Запускается командой `python list_folders.py`.
Запросит логин и пароль от почтового ящика.

## 3. Массовое удаление писем.

Список почтовых ящиков и их пароли надо сохранить в json-файле. Например, `accounts.json`.

Запускается командой `python clean_old.py --accounts accounts.json --days --folders INBOX --dry-run`.

- Параметр `accounts` указывает путь до файла.
- Параметр `days` указывает количество дней. Скрипт работает только с письмами старше этого количества. Значение по умолчанию - 365.
- Параметр `folders` указывает названия папок через запятую, которые надо анализировать. По умолчанию - INBOX.
- Параметр `dry-run` отвечает за имитацию. Если он передан, удаление не произойдет и скрипт просто выведет количество писем, удовлетворяющих остальным параметрам.

### Примеры команд.

- Посчитать количество писем в почтовых ящиках в папке Входящие и Спам старше 365 дней - `python clean_old.py --accounts accounts.json --days 365 --folders INBOX,INBOX.Junk --dry-run`.
- Удалить письма письма старше 365 дней в папке Входящие - `python clean_old.py --accounts accounts.json --days 365 --folders INBOX`.

## 4. Массовая смена паролей.

Запускается командой `python reset_password.py --file results/mails_list/domain.json --dry-run`.

- Параметр `file` указывает путь до файла с массивом почтовых ящиков.
- Параметр `dry-run` отвечает за имитацию. Если он передан, изменение паролей не происходит.

После запуска в файле `results/new_pass_DOMAIN` появится информация.

### Пример сценария работы.

- Запускаем `python mails_list.py --domains domain.ru`, чтобы получить все почтовые ящики.
- Запускаем `python reset_password.py --file results/mails_list/domain.ru.json`, чтобы сбросить пароли.
